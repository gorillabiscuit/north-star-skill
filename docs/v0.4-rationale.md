# v0.4 — generalising the ritual beyond code

Date: 2026-05-29

## What changed

v0.4 generalises the North Star ritual from code-only to any project shape
(code, document, mixed) and from "agent with file access" to "agent in a
plain chat with none". One skill, not two; the interview logic is unchanged.

What is shape-aware:

- **Inputs (Phase 1B.0).** Evidence is whatever artifacts the human has
  produced — commits for a code project, docs/decks/notes for a document
  project, both for mixed. In chat-only mode the agent has no workspace
  access and asks the human to paste the relevant material.
- **Outputs (Phase 4).** `PROJECT.md` for code; the project's primary
  strategy document for a document project (ask if absent); a printed
  copy-paste markdown block for chat-only.
- **Working artifact (the contract).** Three concrete forms of the same
  mechanism: a dotfile in the repo, a visibly-named working doc in the
  document workspace, or the in-conversation reprint block.

What is NOT shape-aware:

- The ritual itself: read first, calibrate via read-back, generate
  candidates with coverage of the read-back's load-bearing nouns,
  re-ground, surface contradictions, converge into four artifacts.
- The four artifacts: Mission, Metric, Companion, Boundary.
- The discipline: one focused question at a time, two candidates minimum,
  name your defaults, retrofit lands as a dated amendment.

## Why one skill, not two

The temptation was to fork into a "north-star-code" and a "north-star-doc"
skill. Resisted because the ritual is identical; only inputs and outputs
differ. Forking would duplicate the interview logic and let the two copies
drift. Branching on shape inside one skill keeps the discipline canonical
and forces the universal phrasing that makes the ritual legible to
non-developers and non-founders.

## Vocabulary

A neutral-vocabulary pass moved business-specific defaults ("moat",
"defensible", "product") into marked alternatives, with the default phrasing
aimed at a reader who is neither a developer nor a founder. The
business-reading framing remains available where it is genuinely the right
register.
